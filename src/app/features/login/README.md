# Login Feature

## Структура

### Компоненты

- **login.component.ts** - Главный компонент страницы входа
- **login.component.html** - Шаблон формы входа


### Сервисы

- **auth.service.ts** - Сервис аутентификации
  - `login(username, password)` - Вход с учетными данными
  - `autoLogin()` - Автоматический вход (admin/admin)
  - `logout()` - Выход из системы
  - `isAuthenticated()` - Проверка авторизации
  - `getToken()` - Получение токена
  - `getCurrentUser()` - Получение текущего пользователя

### Guard'ы

- **auth.guard.ts** - Защита роутов от неавторизованных пользователей

### Interceptors

- **auth.interceptor.ts** - Автоматически добавляет Bearer token ко всем HTTP запросам

### DTO

- **auth.dto.ts** - Типы данных для аутентификации
  - `LoginRequestDTO` - Запрос на вход
  - `LoginResponseDTO` - Ответ сервера
  - `UserAuthDTO` - Данные пользователя

## Функционал

### Форма входа

1. **Поля ввода:**
   - Логин
   - Пароль

2. **Кнопка автоматического входа** (вверху):
   - Выделена градиентным фоном

3. **Кнопка "Войти"**:
   - Отправляет данные на сервер
   - Показывает состояние загрузки

### Роутинг

- `/login` - Страница входа
- `/schedule` - Защищенный роут (требует авторизации)
- `/calendar` - Защищенный роут (требует авторизации)
- `/management` - Защищенный роут (требует авторизации)
- По умолчанию перенаправление на `/login`

### Безопасность

- Токен сохраняется в localStorage
- Автоматическое добавление токена к HTTP запросам
- Защита роутов с помощью authGuard
- Перенаправление неавторизованных пользователей на /login

## API Endpoints

- `POST /api/auth/login/` - Вход в систему

## Использование

### Обычный вход

```typescript
authService.login("username", "password").subscribe({
  next: (response) => {
    // Успешный вход
    // Автоматическая навигация на /schedule
  },
  error: (error) => {
    // Ошибка входа
  },
});
```

### Автоматический вход

```typescript
authService.autoLogin().subscribe({
  next: (response) => {
    // Успешный вход с admin/admin
  },
});
```

### Выход

```typescript
authService.logout();
// Очищает токен и данные пользователя
```
